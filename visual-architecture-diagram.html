The Newspaper Pipeline

Six stages from raw feeds to a published operating artifact. Automation handles infrastructure; judgment handles editorial.

Stage 01
Feed Aggregation
A feed aggregator captures RSS feeds and email newsletters into a single queryable layer. The Readwise Reader API pulls everything that landed in your feed over the past 7 days.
Readwise Reader API RSS + Email
Automated
Stage 02
Source Registry Match
Every feed item is matched against a structured source registry (feed_registry.json) that classifies by name, priority (1/2/3), and editorial family (Macro, GTM, Deals, AI, VSaaS).
feed_registry.json Priority 1/2/3 Family tags
Automated
Stage 03
Score & Deduplicate
Each candidate receives a signal weight (0–100) based on feed priority, word count, and content type. Duplicates are merged. The pool is capped to 50 candidates.
Signal weight 0-100 HIGH / MED / LOW 50 candidates
Automated
Stage 04
Editorial Selection & Synthesis
Apply the editorial rules: hard gate (behavior change in 3–12 months?), balance rule (coverage across families), no duplicates, no vibes. Select 10–12 signals. Synthesize into stances, decisions, and system patches.
Hard gate Balance rule 10-12 signals Stances Decisions System patches
Judgment Layer
Stage 05
Render to HTML
The structured output (JSON blocks + prose) is rendered to a purpose-built HTML artifact with sticky navigation, signal filtering, expandable detail cards, and brand-compliant design tokens.
template.html JSON render contract newspaper.html
Automated
Stage 06
Deploy & Review
The HTML artifact is deployed to the operating dashboard. Published editions accumulate into a stance track record—auditable evidence of where you were right, wrong, and why.
Published artifact Stance track record
Automated
